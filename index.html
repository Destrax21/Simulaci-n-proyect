1. Introducción a la simulación

1.1. Definición e importancia de la simulación en la Ingeniería

La simulación es una técnica que permite imitar el funcionamiento de un proceso o sistema real mediante un modelo computacional. En ingeniería, la simulación es crucial porque:

  • Permite analizar sistemas complejos sin alterar el entorno real
  • Reduce costos al evitar implementaciones físicas prematuras
  • Facilita la identificación de cuellos de botella y puntos de mejora
  • Permite evaluar múltiples escenarios en poco tiempo
Ejemplo Práctico:

Simulación del flujo de tráfico en una intersección para determinar la configuración óptima de semáforos que minimice los tiempos de espera.

1.2. Conceptos básicos de la simulación

Para comprender la simulación, es fundamental dominar estos conceptos:

Concepto Definición
Modelo Representación simplificada de un sistema real
Entidad Elemento que se mueve a través del sistema
Atributo Característica de una entidad
Evento Suceso que cambia el estado del sistema
Recurso Elemento que presta servicio a las entidades

2. Números pseudoaleatorios

2.1. Métodos de generación de números Pseudoaleatorios

Los números pseudoaleatorios son secuencias deterministas que aparentan ser aleatorias. Métodos comunes:

Método de Congruencia Lineal

Generador más común: Xn+1 = (aXn + c) mod m

Donde: a = multiplicador, c = incremento, m = módulo

Método de Fibonacci

Xn = (Xn-1 + Xn-2) mod m

Requiere dos semillas iniciales

Ejemplo en Python:
# Generador de números pseudoaleatorios
def linear_congruential(seed, a=1664525, c=1013904223, m=2**32, n=10):
    results = []
    x = seed
    for _ in range(n):
        x = (a * x + c) % m
        results.append(x / m)  # Normalizar a [0,1)
    return results

print(linear_congruential(seed=12345))
2.2.1. Pruebas de uniformidad

Verifican si los números generados siguen una distribución uniforme:

  • Prueba Chi-cuadrado
  • Prueba de Kolmogorov-Smirnov
  • Prueba de series
2.2.2. Pruebas de aleatoriedad

Las pruebas de aleatoriedad evalúan si una secuencia numérica carece de patrones predecibles:

Prueba de Rachas (Runs Test):

Evalúa las secuencias crecientes y decrecientes en la serie numérica. Calcula:

Z = (R - μ_R) / σ_R

Donde R es el número total de rachas, μ_R y σ_R son la media y desviación esperadas.

2.2.3. Pruebas de Independencia

Evalúan si los números en la secuencia son independientes entre sí:

  • Prueba de Autocorrelación
  • Prueba de Gap
Implementación en Python:
from statsmodels.sandbox.stats.runs import runstest_1samp

# Secuencia de números pseudoaleatorios
data = [0.12, 0.45, 0.78, 0.23, 0.56, 0.89, 0.34, 0.67]

# Aplicar prueba de rachas
z_stat, p_value = runstest_1samp(data)
print(f"Estadístico Z: {z_stat:.4f}, Valor p: {p_value:.4f}")

3. Generación de variables aleatorias

3.1. Distribuciones de probabilidad

Las variables aleatorias en simulación a menudo siguen distribuciones de probabilidad específicas, como:

  • Discretas: Bernoulli, Binomial, Poisson, Geométrica
  • Continuas: Uniforme, Exponencial, Normal, Triangular
Ejemplo: Distribución de Poisson

Describe la probabilidad de un cierto número de eventos que ocurren en un intervalo fijo de tiempo o espacio, si estos eventos ocurren con una tasa media constante e independientemente del tiempo transcurrido desde el último evento.

import numpy as np
import matplotlib.pyplot as plt

# Generar 1000 muestras de una distribución de Poisson con lambda = 5
poisson_data = np.random.poisson(lam=5, size=1000)

plt.hist(poisson_data, bins=range(min(poisson_data), max(poisson_data) + 2), density=True, alpha=0.7, color='skyblue', edgecolor='black')
plt.title('Histograma de Distribución de Poisson')
plt.xlabel('Número de eventos')
plt.ylabel('Frecuencia normalizada')
plt.grid(axis='y', alpha=0.75)
plt.show()

Este código simula eventos raros, como la cantidad de llamadas a un centro de atención en una hora.

3.2. Métodos para generar variables aleatorias

Se utilizan diversos métodos para transformar números pseudoaleatorios uniformes en variables con distribuciones deseadas.

3.2.1. Método de la transformada inversa

Para distribuciones con F(x) invertible:

  1. Generar U ~ Uniforme(0,1)
  2. Calcular X = F⁻¹(U)

Aplicaciones: Exponencial, Weibull, Cauchy

3.2.2. Método de convolución

Para variables que son suma de otras independientes:

X = Y₁ + Y₂ + ... + Yₙ

Ejemplo:

Erlang(k,λ) es suma de k exponenciales(λ)

3.2.3. Método de composición

Para mezclas de distribuciones:

f(x) = Σ pᵢfᵢ(x)

Pasos:

  1. Seleccionar fᵢ con probabilidad pᵢ
  2. Generar X de fᵢ
Ejemplo: Generación de normales por Box-Muller
import math
import random

def box_muller():
    u1 = random.random()
    u2 = random.random()
    z0 = math.sqrt(-2*math.log(u1)) * math.cos(2*math.pi*u2)
    z1 = math.sqrt(-2*math.log(u1)) * math.sin(2*math.pi*u2)
    return z0, z1

Este método transforma dos uniformes en dos normales estándar independientes.

4. Lenguajes de simulación

4.1. Tipos de lenguajes de simulación

Existen diversas herramientas y lenguajes para llevar a cabo proyectos de simulación:

  • Lenguajes de propósito general: Python (SimPy), Java, C++
  • Software de simulación dedicado: AnyLogic, Arena, FlexSim, Simulink
  • Hojas de cálculo: Excel (para simulaciones sencillas)
Ventajas de software dedicado:

AnyLogic, por ejemplo, permite combinar diferentes paradigmas de modelado (eventos discretos, basado en agentes, dinámica de sistemas) en un mismo modelo, lo que lo hace muy versátil para sistemas complejos.

4.2. Selección del software de simulación

La elección de la herramienta depende de varios factores:

  • Complejidad del sistema: ¿Se necesitan múltiples paradigmas de modelado?
  • Nivel de detalle: ¿Se requiere simular a nivel de proceso, de agente individual, o agregado?
  • Presupuesto y licencias: Disponibilidad de licencias comerciales o alternativas de código abierto.
  • Curva de aprendizaje: Facilidad de uso y documentación.
  • Capacidad de análisis: Herramientas integradas para estadística y optimización.
4.3. Casos prácticos de simulación
4.3.1. Problemas con líneas de espera
Modelo M/M/1:

Parámetros clave:

  • λ: Tasa de llegadas (clientes/tiempo)
  • μ: Tasa de servicio (clientes/tiempo)
  • ρ = λ/μ: Intensidad de tráfico

Métricas:

L = ρ/(1-ρ) (N° esperado en sistema)

W = 1/(μ-λ) (Tiempo esperado en sistema)

4.3.2. Problemas con sistemas de Inventarios
Modelo (s,S):

Cuando el inventario baja de s, ordenar hasta S.

Costo total:

CT = Cₕ·I + Cₒ·Nₒ + Cₚ·Dₚ

Donde:

  • Cₕ: Costo de mantener
  • Cₒ: Costo de ordenar
  • Cₚ: Costo por faltante

Caso Práctico

Simulación de un sistema de inventarios
Problema:

Una empresa minorista necesita determinar la política óptima de reabastecimiento para minimizar costos manteniendo un nivel de servicio adecuado.

Parámetros del sistema:
  • Demanda diaria: Distribución normal (μ=50, σ=10)
  • Tiempo de entrega: 2 días
  • Costo de ordenar: $50 por pedido
  • Costo de mantenimiento: $2 por unidad por día
  • Costo por faltante: $10 por unidad
  • Inventario inicial: 150 unidades
Modelado y simulación:

Se implementó un modelo de simulación de eventos discretos para evaluar diferentes políticas (s, S):

# Pseudocódigo para simulación de inventarios
Initialize:
    inventory = 150
    time = 0
    order_cost = 0
    holding_cost = 0
    shortage_cost = 0

Events:
    Customer arrival:
        demand = normal(50, 10)
        if inventory >= demand:
            inventory -= demand
        else:
            shortage = demand - inventory
            shortage_cost += shortage * 10
            inventory = 0

    Order arrival:
        inventory += order_quantity

    Daily review:
        if inventory <= reorder_point:
            place_order(order_up_to_level - inventory)
            order_cost += 50
        holding_cost += inventory * 2
Resultados:

La política óptima encontrada fue (s=100, S=300) con un costo diario promedio de $142.
